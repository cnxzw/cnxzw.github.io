markdown语法简要

上次我简单学习了利用github和hexo搭建个人博客网站，网站搭建好了之后如何写博文成文当务之急。Hexo框架使用Markdown解析文章，在Markdown官网简单学习了一下，现将Markdown的语法做一个简单的记录。

什么是Markdown

Markdown 的目标是实现「易读易写」。

不过最需要强调的便是它的可读性。一份使用Markdown格式撰写的文件应该可以直接以纯文字发布，并且看起来不会像是由许多标签或是格式指令所构成。Markdown语法受到一些既有text-to-HTML格式的影响，包括Setext、atx、Textile、reStructuredText、Grutatext和EtText，然而最大灵感来源其实是纯文字的电子邮件格式。

因此Markdown的语法全由标点符号所组成，并经过严谨慎选，是为了让它们看起来就像所要表达的意思。像是在文字两旁加上星号，看起来就像强调。Markdown的清单看起来，嗯，就是清单。假如你有使用过电子邮件，区块引言看起来就真的像是引用一段文字。

Markdown语法的主要内容

  • 标题
  • 区块
  • 强调
  • 列表
  • 链接
  • 图片
  • 代码

标题

Markdown支持两种标题的语法，Setext和atx形式
==== 高级标题， —– 二级标题
1~6个#，对应1~6级标题

区块

在Markdown文件中简历一个区块，看起来像是强迫断行，需要在每行最前面加上 >

强调

*内容*
**内容**
_内容_
__内容__

列表

列表类似于清单，用于列出单子，无序清单使用 * + - 皆可

链接

博客中需要插入网址等链接
[链接描述](链接地址)

图片

博客中插入图片，和插入链接差不多，只是需要在最前面加上!:
![图片描述](图片所在地址或路径)
需要注意的是在windows系统中路径需要添加\,例如：
E:\me\Blog\blog

代码

如果要标记一小段行内程式码，你可以用反引号把它包起来；如果要在程式码区段内插入反引号，你可以用多个反引号来开启和结束程式码区段，例如：
``` bash
code block
```

以上就是对Markdown语法的简单回顾

利用GitHub与Hexo构建个人博客

为了更好的总结自己的学习收获，本人决定从即日起养成写博客的习惯。首先要解决的就是如何开始自己的博客之旅，本着Geek的精神，在知乎上花了不到一天的时间，参考了吴润的GitHub+Hexo搭建个人网站详细教程，学习了怎么通过GitHub和Hexo框架来搭建自己的博客，下面就是对此方法的记录与总结。

关于Hexo

Hexo是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。网页可以托管在GitHub和Heroku上，我们选用的是GitHub。由于前期主要用于试验（其实就是穷），并没有购买域名，相关内容暂时不做说明。

搭建步骤

1.安装Git
2.安装Node.js
3.安装Hexo
4.推送网站
5.更换主题
6.发布文章

在开始之前首先需要一个GitHub账号，若没有可使用邮箱注册，点击GitHub中的New repository创建新仓库，基于习惯，我们把仓库名设置为：用户名.github.io。

1.安装Git

Git是开源的分布式版本控制系统，用来高效处理项目。从Git官网 下载合适的安装包，然后进行安装，在命令行里输入git查看安装是否成功。成功安装后，鼠标右击运行Git Bash，设置user.name和user.email

生成ssh密钥文件

一路回车，不需要设置密码，找到生成的.ssh的文件夹中的id_rsa.pub密钥，复制所有内容，打开GitHub_Settings_keys ，新建new SSH Key,将复制内容粘贴在Key,点击Add SSH Key .
在Git Bash中输入 ssh git@github.com 检测是否设置成功 ，出现“You’ve successfully authenticated”字样说明设置成功。

安装Node.js

由于Hexo基于Node.js, 所以需下载安装Node.js.
检测Node.js是否安装成功，在命令行中输入 node -v;检测npm是否安装成功，输入 npm -v。

安装Hexo

到这我们需要安装Hexo，首先在电脑里创建一个文件夹，即将来博客所在位置，进入文件夹按住shift键，右键选择打开命令行，使用npm安装Hexo

安装完成后初始化博客：

这里简单介绍以下主要的Hexo命令

npm install hexo -g #安装Hexo
npm update hexo -g #升级
hexo init #初始化博客

hexo n “Blog Name” == Hexo new “Blog Name” #新建文章
hexo g == hexo generate #生成
hexo s == hexo server #用于本地预览，localhost:4000
hexo d == hexo deploy #部署
hexo clean #清楚缓存

一般情况下，我们在写好文本后，分别输入三条命令即可：

打开浏览器，在地址栏输入 xxxx.github.io就会发现写的博客可以在网络上访问了。
